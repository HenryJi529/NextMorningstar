## 为什么

API 业务错误当前使用 `console.log` 输出，会让异常响应看起来像普通调试信息。将这些响应按错误级别记录，可以继续暴露隐藏的后端或集成问题，同时让日志严重程度更准确。

## 变更内容

- 将 API 响应拦截器中非成功业务码的日志从 `console.log` 改为 `console.error`。
- 保持现有 token 无效 / 过期处理逻辑不变。
- 保持现有网络错误的用户提示逻辑不变。

## 能力

### 新增能力
- `api-error-observability`：业务级 API 失败会以错误级别输出到浏览器控制台。

## 影响范围
