## 上下文

前端 Axios 响应拦截器已经会在 `response.data.code <= 0` 时识别业务级 API 失败。当前这些失败使用 `console.log` 输出，而传输层错误仍通过现有 Ant Design 消息向用户提示。

## 目标 / 非目标

**目标：**
- 在浏览器控制台中用错误级别记录业务级 API 失败。
- 保持现有响应流程和 token 无效 / 过期处理逻辑不变。

**非目标：**
- 不为业务级 API 失败新增用户可见提示。
- 不修改网络错误或服务端传输错误处理逻辑。
- 不修改 API 响应结构或成功码语义。

## 决策

### 决策 1：只调整控制台日志级别

对 `response.data.code <= 0` 使用 `console.error`，替代 `console.log`。这样可以让开发者继续看到隐藏的后端或集成问题，同时不改变运行时控制流，也不向用户展示额外 UI 提示。
